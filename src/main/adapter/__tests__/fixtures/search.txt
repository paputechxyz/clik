Searches the public (logged-out) item board and ingests results through
the same pipeline as 'recommended'. Works without a session; no login needed.
--top N caps the number of items processed end-to-end (detail fetch + LLM score).
Items failing any active user gate (--remote/--hybrid/--min-value) are dropped
in-memory after the detail fetch and never stored or scored.

Examples:
  myapp search "foo bar" arg2 --min-value 200k
  myapp search "a b" "c d" --top 3
  myapp search "foo bar" arg2 --min-value 160k --value-currency CAD

Usage:
  myapp search [keywords] [location] [flags]

Flags:
      --force-overwrite          re-parse and re-score items already in the DB (bypass dedup; overwrites existing values)
  -h, --help                     help for search
      --hybrid                   only keep hybrid-friendly items (combine with --remote for OR)
      --min-value string         only keep items at or above this value (e.g. 200k)
      --no-detail                skip detail page fetching
      --no-filter                skip the hard preference filter
      --no-score                 skip LLM enrichment+fit-scoring
      --remote                   only keep remote-friendly items
      --value-currency string    currency for --min-value (ISO 4217, e.g. CAD); enables FX-aware filtering
      --top int                  cap on number of items to fetch + process end-to-end (default 25)

Global Flags:
      --db string   path to the SQLite DB file (default: ~/myapp/myapp.db or $MYAPP_DB_PATH)
      --json        emit machine-readable JSON (agent-native output)
